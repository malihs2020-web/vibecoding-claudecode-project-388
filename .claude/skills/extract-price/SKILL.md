# Extract price

Извлекает цену товара. Источники:

1. **URL страницы** — одна или несколько прямых ссылок на карточки товара (даны пользователем).
2. **Ozon**
3. **Яндекс Маркет**
4. **Wildberries**

Если готового списка ссылок нет, список ссылок для источника 1 подбирается автоматически через fallback-поиск сторонних магазинов (см. ниже) — источники 2–4 в этот список не входят, они обрабатываются отдельно.

## Шаг 0: определение стратегии

Что именно проверять, зависит от двух независимых факторов:

- **A — дан ли готовый список URL** на карточки товара (одна или несколько ссылок от пользователя).
- **B — доступен ли браузер.** Проверяется вызовом `tabs_context_mcp` (при необходимости с `createIfEmpty: true`): успешный ответ со списком вкладок → браузер **доступен**; ошибка («Browser extension is not connected» и т.п.) → браузер **недоступен**.

Правила (действуют независимо друг от друга и складываются):

| A: список URL | B: браузер | Что делать |
|---|---|---|
| Есть | Есть | Обработать список URL («Порядок действий: URL страницы») **и** дополнительно Ozon / Яндекс Маркет / Wildberries («Порядок действий: Ozon/ЯМ/WB»). |
| Есть | Нет | Обработать только список URL. Ozon / Яндекс Маркет / Wildberries не проверяются. |
| Нет | Есть | Подобрать список URL через fallback-поиск сторонних магазинов, обработать его как список URL **и** дополнительно Ozon / Яндекс Маркет / Wildberries. |
| Нет | Нет | Подобрать список URL через fallback-поиск сторонних магазинов и обработать его как список URL. Ozon / Яндекс Маркет / Wildberries не проверяются. |

Короче: Ozon/ЯМ/WB проверяются всегда, когда браузер доступен (эти сайты блокируют прямые запросы — без браузера к ним не подступиться). Fallback-поиск сторонних магазинов запускается всегда, когда готового списка ссылок нет (независимо от браузера) — это единственный способ получить хоть какие-то данные без браузера.

## Выход

Одна ссылка на входе → один объект:

```json
{ "regular_price": 1990, "sale_price": null }
```

Список ссылок (данный пользователем или подобранный fallback-поиском), а также источники Ozon / Яндекс Маркет / Wildberries → список объектов, по одному на источник/карточку:

```json
{ "name": "...", "url": "...", "regular_price": 1990, "sale_price": null }
```

- `regular_price` — обычная (полная) цена товара, число.
- `sale_price` — цена по скидке/акции, число, или `null`, если скидки нет.

Если по конкретной ссылке/источнику цену найти не удалось — не включать её в список результата, а отдельно отметить как «не удалось получить цену» (с причиной: блокировка, нет в наличии, страница не парсится и т.п.), не подставляя вместо цены данные из сниппетов поиска. Если вообще ничего не найдено — вернуть текстовый ответ: `товар не найден`.

## Общая логика извлечения цены (для всех источников)

1. Загрузить HTML/содержимое страницы.
2. При необходимости распарсить (найти элементы с ценой по типичным признакам: атрибуты/классы вида `price`, `regular-price`, `sale-price`, `old-price`, микроразметка `itemprop="price"`, `<meta property="product:price:amount">` и т.п.).
3. Убрать пробелы, неразрывные пробелы, символы валют и разделители тысяч, привести к числовому виду (точка как разделитель дробной части).
4. Определить, какая цена обычная, а какая — со скидкой:
   - одна цена на странице/в карточке — это `regular_price`, `sale_price` = `null`;
   - две цены (старая/зачёркнутая и новая) — старая идёт в `regular_price`, новая — в `sale_price`.
5. Если цену найти не удалось — `товар не найден` (для этой ссылки/карточки).

## Общее правило точного совпадения (для поисковых источников: fallback-поиск, Ozon, Яндекс Маркет, Wildberries)

Поиск/сортировка по цене смешивает разные модификации товара (другой вкус, вес, размер/порода, объём упаковки, другой магазин с похожим, но не тем же товаром). Поэтому среди найденных/отсортированных по возрастанию цены вариантов нужно оставлять для сравнения только те, чьё название/характеристики **точно** соответствуют запрошенному товару — даже если менее подходящий вариант дешевле, его нужно пропустить. Если характеристика не даёт однозначно понять соответствие по тексту (например, вес), проверять по изображению упаковки (zoom скриншота карточки) или по описанию на странице товара.

## Порядок действий: URL страницы

Применяется к каждой ссылке из списка (готового или подобранного fallback-поиском) по отдельности.

1. Загрузить HTML страницы по переданному URL (WebFetch; если сайт блокирует прямые запросы и доступен браузер — открыть через браузер).
2. Применить общую логику извлечения цены.
3. Если ссылок несколько — собрать результаты в список объектов `{ name, url, regular_price, sale_price }` по формату «Выход»; если ссылка одна — вернуть один объект.

## Порядок действий: fallback-поиск сторонних магазинов (когда готового списка URL нет)

Цель — набрать 5–10 рабочих ссылок на сторонние интернет-магазины (НЕ Ozon, НЕ Яндекс Маркет, НЕ Wildberries — эти три при отсутствии браузера всё равно не спарсятся), по которым реально получается вытащить цену через `WebFetch`.

1. Вызвать `WebSearch` по названию/характеристикам товара, исключив домены `ozon.ru`, `market.yandex.ru`, `wildberries.ru` через `blocked_domains`.
2. Из результатов отобрать кандидатов — ссылки на конкретные карточки товара (не категории/обзоры/агрегаторы отзывов), по возможности разных магазинов.
3. Для каждого кандидата сделать пробный `WebFetch` с запросом на извлечение названия и цены.
4. Оставить только те ссылки, где `WebFetch` реально вернул читаемую цену — отсеять случаи блокировки (ошибка запроса, HTTP 4xx/498), пустого/усечённого контента и страниц, где цена подгружается через JS и не видна в ответе.
5. Применить общее правило точного совпадения к оставшимся результатам.
6. Если рабочих источников меньше 5 — при необходимости повторить `WebSearch` с уточнённым запросом, чтобы добрать до 5–10. Если после разумного числа попыток рабочих источников нет вообще — сообщить пользователю, что не удалось найти сторонние магазины с доступной для парсинга ценой, и предложить прислать конкретные ссылки или подключить браузер.
7. Итоговый список рабочих URL передать в «Порядок действий: URL страницы» и оформить результат по формату «Выход».

## Порядок действий: Ozon

Ozon блокирует прямые запросы (WebFetch/curl получают петлю редиректов через анти-бот защиту) — работать можно только через реальный браузер (Claude in Chrome).

1. Открыть ozon.ru, ввести в поиск название товара (бренд + категория + ключевые характеристики).
2. Применить доступные фильтры под запрошенные характеристики: бренд, вес (например, диапазон 10000–20000 г для «10 кг»), и т.п.
3. Включить сортировку «Дешевле» (в URL параметр `sorting=price`).
4. Считать список карточек (`get_page_text` и/или `read_page`): название, цена, наличие зачёркнутой (старой) цены.
5. Применить общее правило точного совпадения — отфильтровать карточки, не соответствующие запрошенным характеристикам.
6. Взять топ-N самых дешёвых точных совпадений. Для каждого получить прямую ссылку на карточку (`find` по названию/цене → `read_page` по `ref_id` → атрибут `href`).
7. Оформить результат по формату «Выход» выше.

## Порядок действий: Яндекс Маркет

Тоже требует браузера — WebFetch/curl аналогично блокируются или не отдают цену (JS-рендеринг).

1. Открыть `market.yandex.ru/search?text=<название товара>`.
2. При необходимости задать фильтры в панели слева (цена, бренд и т.п.).
3. Отдельного надёжного переключателя «сортировка по цене» на карточках маркетплейса может не быть — считывать список карточек через `get_page_text` (список подгружается частями, при необходимости повторять запрос / прокручивать страницу).
4. Применить общее правило точного совпадения.
5. Цена на карточке обычно в формате «Цена с картой Яндекс Пэй X ₽ вместо Y ₽»: `X` — это `sale_price` (цена с картой/акцией), `Y` — `regular_price`. Если указана только одна цена — это `regular_price`, `sale_price` = `null`.
6. Взять топ-N самых дешёвых точных совпадений, получить ссылку карточки (`find` → `read_page` по `ref_id` → `href`, ведёт на `/card/...`).
7. Оформить результат по формату «Выход» выше.

## Порядок действий: Wildberries

Тоже требует браузера.

1. Открыть `wildberries.ru/catalog/0/search.aspx?search=<название товара>`.
2. Открыть переключатель сортировки («По популярности») и выбрать «По возрастанию цены» (в URL добавляется `sort=priceup`).
3. Так как отдельного фильтра «вес» может не быть, а сортировка по цене смешивает разные объёмы упаковки (1 кг, 3 кг, 10 кг и т.д.) — задать в «Все фильтры» диапазон цены, отсекающий заведомо не подходящие по объёму варианты (например, для 10 кг корма — нижнюю границу в районе цены за большую упаковку), либо проверять вес по каждой карточке.
4. Для карточек, где вес/вкус не виден в текстовом названии, — увеличить (zoom) скриншот карточки, чтобы прочитать вес/вкус с изображения упаковки.
5. Применить общее правило точного совпадения.
6. Взять топ-N самых дешёвых точных совпадений, получить ссылку карточки (`find` → `read_page` по `ref_id` → `href`, ведёт на `.../detail.aspx`).
7. Оформить результат по формату «Выход» выше.
