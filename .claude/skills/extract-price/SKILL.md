# Extract price

Извлекает цену товара со страницы интернет-магазина по её URL. Не занимается поиском/подбором источников — ссылки должны быть уже даны на входе (пользователем или другим skill'ом, например `tracker`).

## Вход

Один URL страницы товара, либо список URL.

## Выход

Один URL на входе → один объект:

```json
{ "regular_price": 1990, "sale_price": null }
```

Список URL на входе → список объектов, по одному на URL, где цену удалось извлечь:

```json
{ "name": "...", "url": "...", "regular_price": 1990, "sale_price": null }
```

- `regular_price` — обычная (полная) цена товара, число.
- `sale_price` — цена по скидке/акции, число, или `null`, если скидки нет.

Если по конкретному URL цену найти не удалось — не включать его в список результата, а отдельно отметить как «не удалось получить цену» с причиной (блокировка, страница не парсится, товара нет в наличии и т.п.), не подставляя вместо цены данные из сниппетов поиска или другие догадки. Если URL был один и цена не найдена — вернуть текстовый ответ: `товар не найден`.

## Порядок действий

Применяется к каждому URL по отдельности.

1. Загрузить HTML/содержимое страницы по URL (WebFetch; если сайт блокирует прямые запросы и доступен браузер — открыть через браузер).
2. При необходимости распарсить (найти элементы с ценой по типичным признакам: атрибуты/классы вида `price`, `regular-price`, `sale-price`, `old-price`, микроразметка `itemprop="price"`, `<meta property="product:price:amount">` и т.п.).
3. Убрать пробелы, неразрывные пробелы, символы валют и разделители тысяч, привести к числовому виду (точка как разделитель дробной части).
4. Определить, какая цена обычная, а какая — со скидкой:
   - одна цена на странице — это `regular_price`, `sale_price` = `null`;
   - две цены (старая/зачёркнутая и новая) — старая идёт в `regular_price`, новая — в `sale_price`.
5. Если цену найти не удалось — `товар не найден` для этого URL.
6. Оформить результат(ы) по формату «Выход».
